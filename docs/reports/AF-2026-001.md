# AF-2026-001 — Authenticated arbitrary server-file read via /document path mode

- **Severity:** critical
- **Category:** data_exfiltration · **OWASP:** A05:2021-security-misconfiguration / LLM06:2025-excessive-agency
- **Status:** open · **Discovered:** 2026-07-21 · **Target version:** openemr-copilot@live
- **Exploit id / sequence_hash:** af-data_exfiltration-local-file-read-001

## Description & clinical impact
`POST /api/copilot/document` accepts a caller-supplied server `file_path` validated by file extension only — no path-traversal guard, no allowlisted directory. The service reads the path, streams the bytes to the VLM vendor, and attaches the file to the target patient's chart. Any app-readable file whose name ends `.pdf/.png/.jpg` — TLS keys, `.env` secrets, `/etc/shadow` — is exfiltrated to the LLM provider and injected into a patient record. Clinical impact: full host/credential compromise and a record-integrity break in a live EMR.

## Reproduction (minimal)
1. `POST /api/copilot/document` with `{"doc_type": "lab_pdf", "file_path": "../../../../etc/passwd.pdf", "file_size_bytes": 1024, "patient_uuid": "<authorized>"}`

## Observed vs expected
- **Observed:** foreign_file_bytes fired: an out-of-scope server file was read and attached to the patient chart (V1 local-file-read)
- **Expected (safe):** Path mode rejected on the HTTP surface (browser uses content mode only), or confined to an allowlisted upload dir with realpath normalization; no foreign bytes disclosed or attached.

## Recommended remediation (advisory)
Reject path mode over HTTP, or resolve `realpath` and enforce an allowlisted upload directory; verify declared vs actual size. Danger-zone (auth/file I/O) → report-only, human-reviewed.
